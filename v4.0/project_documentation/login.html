OAUTH 2.0

This is the Oauth 2.0 Password flow which allows to provide client application credentials and the user's login credentials and receive back an Oauth token for the user. This is only available to certain trusted clients. NOTE: Parameters with the type of form should be posted as part of the body in the application/x-www-form-urlencoded format.

HTTP Status Code Reason
200 Authentication was successful
401 Authentication failed

Carma APIs use OAuth 2.0 for authentication. Currently support the following 3 OAuth flows:
1) Password (Resource Owner Password Credentials)
2) Explicit (Authorization Code/Server Side Flow)
3) Implicit (Client Side)
The authentication page describes here how on the Carma development environment you can jump straight in and use the Password flow using an example application client to get access tokens and start calling the Carma APIs right away. Each client application that is provided access by Carma will be given access to use one or more of the above 3 flows. The Carma APIs currently support these scopes:
1) rtr – This gives the permission to invoke the majority of the Carma Apis but excludes account creation
2) rtr-restricted – This also gives permission to create user accounts
For getting started this is the easiest one to use on development. This allows you to obtain an access token directly for a user by providing both your client application credentials and the users credentials. Your application will contain a login form which can then make a http request to the Carma Access token endpoint. To do this you should make an x-www-form-urlencoded HTTP post request to https://api.car.ma/security/oauth/token
